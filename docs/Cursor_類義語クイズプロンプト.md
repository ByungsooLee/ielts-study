# Cursor 実装プロンプト：類義語クイズ（覚えやすい構成・干渉を避ける設計）

> 既存アプリに「類義語クイズ」モードを追加する指示。作り直さず統合。まず現状把握→計画→承認→実装。
> 教材データ・保管は既存方針どおり（KV `content` から取得、`StudyItem.synonyms` を使用）。スキーマ変更は不要。

## 設計の根拠（語彙習得研究）
- **未知の類義語を同時に大量提示すると相互干渉**で記憶が落ちる（最も干渉が強い意味関係が synonym）。→ **新規の類義語を一度に詰め込まない**。
- 効果的なのは、**すでに知っている見出し語を核に**・**文脈（使い方）の中で**・**間隔をあけて**・**自力で思い出す（retrieval）**形。
- 近い語の弁別は、各語が個別に分かってから**識別タスク**で行う。

→ 実装方針：①出題対象は「ある程度学習済みの見出し語」に限定。②文脈穴埋めを中核に。③識別は後段。④SRSで間隔をあけ、同じ類義語クラスタを連続提示しない。

## データ（既存を使う・変更不要）
- 各 `StudyItem` の `synonyms: string[]`（その語の言い換え群）。
- `examples[].en`（文脈穴埋めの土台）。
- 誤答プール：他項目の `front`/`synonyms`（**意味が無関係**の語を選ぶ）。同 `type` 優先で品詞感を揃える。

## 出題対象のゲート（干渉回避の要）
- 類義語クイズに出すのは **`synonyms` を持ち、かつ見出し語が学習済みの項目**（例：SRSで `reps>=1` または「学習済み」）。
- 未学習語は通常学習で先に覚えてから類義語へ。新しい類義語クラスタは**1セッションに少数ずつ**。

## クイズ形式（覚えやすさ順に4種。設定でON/OFF可）
### A. 言い換え選択（MCQ・入口）
- 提示：見出し語＋意味。「同じ意味はどれ？」→ 選択肢4つ（正解＝その語の `synonyms` から1つ、誤答3つ＝無関係プール）。
- 役割：認識の入口。速い。

### B. パラフレーズ穴埋め（中核・IELTS直結）
- 提示：その語を含む `example.en` の語を下線/空所化。「下線の語と置き換えても意味が通るのは？」→ 選択肢（正解＝`synonyms`、誤答＝無関係語）。
- 文脈で意味・使い方を学ぶ。**Writingの言い換え力**に直結。
- 注意：類義語は完全互換ではないため、**選択肢に“同じ語の別の類義語”を誤答として混ぜない**（誤誘導になる）。誤答は意味の遠い語にする。

### C. 仲間外れ（識別）
- 4語のうち3語が同義群、1語だけ別。→ 仲間外れを選ぶ。近い語の弁別を鍛える。
- 既に各語を学習済みの場合のみ出題。

### D. 産出タイプ（最強の想起）
- 提示：意味＋（任意で）文脈。「言い換えを1語で入力」→ その語の `synonyms`（および見出し語自身）を正解として許容。先頭文字ヒント可。
- 自力産出。スペル練習にもなる。

## 採点・SRS・干渉回避
- 3段階採点（忘れた／あいまい／覚えてた）でその項目の復習間隔に反映（既存SM-2を流用）。
- **同じ類義語クラスタを連続で出さない**（最低数問あける）。1セッションの新規類義語は少数に制限（既定 5）。
- 苦手（lapses多）の語は出題頻度を上げる。
- 発音：選択肢・正解語は🔊で発音確認できる（既存TTS/辞書を流用）。

## UI
- 既存のカード/カテゴリUIに「類義語」モードを追加（常設ボタン or 学習モード選択に併設）。
- 1問1カード・即時フィードバック（正誤＋正解の言い換え一覧＋例文）。モバイル対応・キーボード（1〜4で選択、Enter）。
- セット量（10/30/50）やテーマ範囲フィルタは既存設定を流用。

## 厳守
- 既存スタック・データ取得（KV `content`）・SRS・発音を壊さない。新ライブラリは理由を添えて確認。
- 誤答生成は「意味が遠い語」を選ぶこと（近すぎる語は出題者側で除外）。

## 進め方
1. 現状把握（既存の出題・SRS・データ取得層）＋統合計画を提示し承認。
2. 出題対象ゲート＋誤答プール生成のユーティリティ。
3. 形式A・B（中核）→ C・D を順に。各完了で動作確認手順。
4. SRS連携・干渉回避（連続回避・新規上限）・発音・モバイル調整。

## 受け入れ基準
1. 学習済みかつ `synonyms` を持つ語だけが出題される（未学習の類義語を大量に初出させない）。
2. A〜Dが動作し、正解＝その語の `synonyms`、誤答＝意味の遠い語になっている。
3. パラフレーズ穴埋めで、選択肢に“同義の別語”を誤答として混ぜていない。
4. 採点が既存SRSに反映され、同一クラスタが連続出題されない。
5. 発音確認ができ、モバイル/キーボード操作に対応。既存機能のリグレッションなし。
