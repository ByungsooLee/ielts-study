# Cursor 実装プロンプト：テーマ番号で選ぶボタンUI（単語）

> 既存アプリに「テーマ番号で単語を選べるボタン」を追加。作り直さず統合。まず現状把握→計画→承認→実装。
> データは取得済みの `StudyItem`（`theme`=テーマ番号, `themeName`=日本語名, `tags`）を使用。スキーマ変更不要。

## 目的
単語学習で、**テーマを番号ボタンでワンタップ選択**して、その範囲の語だけを出題できるようにする。
（例：1〜10 のボタンが並び、押すとそのテーマの単語に絞り込まれる）

## データ前提
- 各 item は `theme`(整数, 例 1..10) と `themeName`(例「タイタニック」) を持つ。`theme` の無い項目（文法3件など）は番号ボタンに含めない。
- テーマ番号・名称・件数は**データから動的に算出**（ハードコード禁止）。テーマは今後増える前提。

## 仕様
### ボタン群（テーマセレクター）
- データに存在する `theme` を**昇順に番号ボタン**として表示（`1 2 3 … 10`）。各ボタンは数字を大きく、下に `themeName` を小さく（または長押し/ホバーでツールチップ）。
- 任意で各ボタンに**件数バッジ**（そのテーマの単語数）。
- **「全部」ボタン**（全テーマ）も用意。既定は「全部」または前回選択（localStorage 保存）。
- 選択は**単一選択**（1テーマ）。選択中を明確にハイライト。
  - 任意拡張：複数選択（チップ式）も可。実装するなら設定で切替。

### テーマが増えたとき（10刻みレンジ）
- テーマが11以上になったら、上段に**レンジ切替（`1–10` `11–20` …）**を出し、選んだレンジの番号ボタンだけを下段に表示（`item.theme` から動的生成）。
- スマホは番号ボタンを横スクロール or グリッド折返し。ハンバーガー内にも同じセレクターを置く。

### 絞り込みの連携
- 選んだテーマは**単語カテゴリ（type=word）の出題プール**を絞る。既存の「セット量(10/30/50)」「並び(ランダム/昇順)」「SRS/今日の復習」「文脈モード」とすべて併用できる（テーマ→量→並びの順で適用）。
- カテゴリ（単語/構文/文法/会話）が別にある場合、このテーマ番号ボタンは少なくとも**単語**で機能すること（フレーズにも流用可）。文法（テーマなし）は番号ボタン対象外。

### 表示・操作
- レスポンシブ（スマホ/iPad/PC）。ボタンは押しやすいサイズ、選択状態が一目で分かる配色。
- キーボード：数字キーで該当テーマ選択、矢印で移動可。`aria-pressed` で選択状態を表現。
- 選択テーマと件数を見出しに表示（例「テーマ3 タイタニック・22語」）。

## 厳守
- 既存スタック・データ取得（KV `content`）・SRS・発音・文脈モードを壊さない。
- テーマ番号/名称/件数はデータ駆動。新ライブラリは理由を添えて確認。モバイルファースト・a11y配慮。

## 進め方
1. 現状把握（既存のフィルタ/テーマ表示、状態管理、データ取得層）＋統合計画を提示し承認。
2. テーマ算出ユーティリティ（data → {theme, themeName, count}[]）。
3. 番号ボタンUI（単一選択・全部・ハイライト・件数バッジ）。
4. レンジ切替（11以上で出現）＋スマホ/ハンバーガー対応。
5. 既存の量/並び/SRS/文脈モードとの連携、localStorage保存、キーボード/a11y。

## 受け入れ基準
1. データにある全テーマが番号ボタンで並び、`themeName` と件数が分かる。
2. 番号を押すとその単語だけに絞り込まれ、量(10/30/50)・並び(ランダム/昇順)と併用できる。
3. テーマが11以上になると `1–10 / 11–20` のレンジ切替が現れ、動的に番号が出る。
4. 選択状態が保存され、スマホ/PCで操作でき、キーボード/ariaに対応。
5. 文法（テーマなし）は番号対象外。既存機能のリグレッションなし。
