# ielts-study システム構成

個人用の **英語 + エンジニアリング学習アプリ**。フロントは **Cloudflare Pages**、API は **Cloudflare Worker + KV**、教材は **静的 JSON（CDN）**、端末側は **IndexedDB / localStorage** で動作する。

## 本番 URL（例）

| 役割 | URL |
|------|-----|
| アプリ | `https://ielts-study.pages.dev` |
| API | `https://ielts-study-worker.byung4050.workers.dev` |

## 3. 起動時のデータフロー

```mermaid
sequenceDiagram
  participant U as ユーザー端末
  participant P as Pages (CDN)
  participant W as Worker
  participant IDB as IndexedDB
  participant LS as localStorage

  U->>P: GET /content/index.json
  Note over U,P: 目次だけ先に取得

  U->>LS: 合言葉（env / 設定画面）・設定を読込
  Note over U,LS: 合言葉は手動設定。Worker からは自動取得しない

  U->>IDB: ローカル教材・shardMeta 読込

  alt 同期設定あり
    U->>W: GET /content（KV・レガシー）
    U->>W: GET /progress
    Note over U,W: ローカルとマージ
    U->>IDB: 教材 upsert
    U->>LS: 進捗保存
  end

  Note over U,P: テーマ/ジャンル選択時
  U->>P: GET /content/.../theme-N.json
  Note over U,P: version 差分時のみ再取得
  U->>IDB: replaceThemeContent + shardMeta
```

### 教材取得方針（現行）

1. **起動時** → `index.json` のみ取得
2. **テーマ選択** → 該当シャードを lazy 取得
3. **`version` 差分** → 内容 MD5（8桁）が変わったときだけ再ダウンロード
4. **KV `/content`** → 起動時マージ用のフォールバック（静的 CDN への移行中）

## 4. Worker API

```mermaid
flowchart LR
  CLIENT["Web App"]

  CLIENT -->|"GET /app-bootstrap"| BOOT["公開メタ情報<br/>apiVersion / authMode /<br/>requiresToken / features<br/>認証不要・秘密情報なし"]
  CLIENT -->|"Bearer Token（env/設定）"| AUTH{認証 OK?}

  AUTH -->|Yes| C["GET/PUT /content"]
  AUTH -->|Yes| PR["GET/PUT /progress"]
  AUTH -->|Yes| TTS["POST /tts"]
  AUTH -->|Yes| USAGE["GET /tts-usage"]
  AUTH -->|No| UNAUTH["401 Unauthorized"]

  C --> KV_C["KV: content"]
  PR --> KV_P["KV: progress"]
  TTS --> GOOGLE["Google TTS API"]
  TTS --> KV_U["KV: TTS使用量"]
```

| エンドポイント | 認証 | 用途 |
|---|---|---|
| `GET /app-bootstrap` | 不要 | 公開メタ情報（`apiVersion` / `authMode` / `requiresToken` / `features`）のみ返す。`SYNC_TOKEN` 等の秘密情報は返さない |
| `GET/PUT /content` | Bearer | KV 教材の取得・保存（レガシー） |
| `GET/PUT /progress` | Bearer | 学習進捗（SRS 等）の同期 |
| `POST /tts` | Bearer | Google TTS プロキシ（MP3 返却） |
| `GET /tts-usage` | Bearer | 月次 TTS 使用量 |

## 5. 教材データモデル

```mermaid
flowchart TB
  INDEX["index.json<br/>collections[]"]
  INDEX --> COL1["collection: ielts-vocab<br/>domain: english"]
  INDEX --> COL2["collection: grammar<br/>domain: english"]
  INDEX --> COL3["collection: system-design<br/>domain: engineering"]
  INDEX --> COLN["... 他コレクション"]

  COL1 --> T1["theme-1.json<br/>items + passage"]
  COL2 --> G1["genre-1.json<br/>items"]
  COL3 --> E1["theme-1.json<br/>items + diagram?"]

  T1 --> ITEM["item<br/>id, front, meaning,<br/>examples, pron, explain..."]
  E1 --> ITEM2["item + diagram.mermaid<br/>explain.targets, phrases"]
```

**階層:** `domain` → `collection` → `theme` → `item`

| domain | 例 | 主な item 種別 |
|---|---|---|
| `english` | ielts-vocab, grammar | word, phrase, grammar + passage（長文） |
| `engineering` | system-design, sql-optimization 等 | concept（図解 + 英語説明ドリル） |

### 静的配信レイアウト

```
web/public/content/
  index.json                          ← 目次（collections → themes）
  english/ielts-vocab/theme-0..10.json
  english/grammar/genre-1..3.json
  engineering/system-design/theme-1.json
  engineering/.../theme-1.json
```

- `index.json` の各 theme エントリに `file`, `count`, `version` を保持
- `version` は内容ハッシュ（MD5 先頭8桁）。中身が変わらなければ再取得しない

```mermaid
flowchart LR

  subgraph LocalStorage["localStorage (ielts:*)"]
    PR["ielts:progress<br/>SRS・streak"]
    SET["ielts:settings<br/>表示・新規上限・アクセント"]
    TOK["ielts:sync-token / worker-url<br/>合言葉（手動）・URL"]

  end

  subgraph Cloud["クラウド"]
    KVP["KV progress"]
    KVC["KV content（レガシー）"]

  C <-->|フォールバック| KVC
```

| 保存先 | データ | 端末間同期 |
|---|---|---|
| IndexedDB | 教材、shardMeta、TTS キャッシュ、録音 | 教材は KV フォールバックのみ |
| localStorage | 進捗（SRS）、設定、合言葉（手動）、端末ID（`ielts:*`） | 進捗は Worker KV `/progress` とマージ |
| KV | content, progress, TTS 使用量 | クラウド正 |

## 8. 教材ビルドパイプライン

```mermaid
flowchart LR
  SRC["content-src/<br/>単語マスター.md<br/>ielts-vocab-data.js<br/>grammar-items.json<br/>collections/*.json<br/>number-ledger.json"]
  BUILD["npm run content:build<br/>tools/md2json.js"]
  OUT1["web/public/content/<br/>index + シャード"]
  OUT2["sample/ielts-import.json<br/>全件1ファイル"]
  PUSH["npm run content:push<br/>→ KV /content"]

  SRC --> BUILD
  BUILD --> OUT1
  BUILD --> OUT2
  OUT2 --> PUSH
```

| コマンド | 作用 |
|---|---|
| `npm run content:build` | `content-src/` → 静的 JSON + sample 生成 |
| `npm run content:push` | sample を Worker KV `/content` に PUT（要 `SYNC_TOKEN`） |
| `git push origin main` | Pages + Worker を CI 自動デプロイ |

- **本番 UI の主経路:** Pages CDN の静的 JSON
- **KV push:** レガシー互換・端末間教材マージ用
