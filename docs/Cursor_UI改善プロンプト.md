# Cursor 実装プロンプト：単語帳UIへの刷新（記憶に残る・発音重視・レスポンシブ）

> 既存アプリ（https://ielts-study.pages.dev/ ・取り込みは `ielts-import.json`）の**UI/UXを刷新**する指示。作り直しではなく既存コードに統合。まず現状を調べ、計画を出してから着手。学習効果の高いUIに作り替える。

---

あなたは「日本語ネイティブのデータエンジニアをオーストラリア就職へ導くプロ」の意図で、**単語ベースの学習・復習が最も記憶に残るUI**を実装します。対象は既存の学習アプリ。**大規模リファクタ禁止／既存機能・データ取り込みは壊さない**。

## 確定事項（再質問不要）
- **テーマ**：取り込みJSONは `theme`（数値）＋`themeName`（日本語）を持つ最新版を使う。`theme` 無し（文法等）は「その他」。※既存項目へのtheme反映のため、初回だけ累積版 `ielts-import.json` を全取り込みでバックフィル、その後は最新差分のみ。
- **モード**：「今日の復習(SRS due優先)」と「セット学習(10/30/50)」を**別モード**に分ける。セット学習側に「今日の復習のみ／苦手だけ／未学習だけ」の重ねトグルを用意。
- **生成効果**：AI添削は使わない。答え面に**自作例文メモ（保存のみ・添削なし）**を置く。録音機能があれば答え面に**録音パネルも併設**（産出＋発音練習）。

## ステップ0：現状把握（最初に実施し報告）
スタック・状態管理・保存方式・現在の画面構成・既存の取り込み/復習/発音機能を調べ、「どこを差し替え／どこを流用」する統合計画を提示し承認を得る。

## データ前提（取り込みJSONに以下が入っている）
`StudyItem` に学習用フィールドがある。特に **`type`（word/phrase/grammar/conversation）**、**`theme`（テーマ番号 1,2,3…）**、**`themeName`（テーマ名・日本語）**、`front`/`meaning`/`synonyms`/`collocation`/`examples[{en,jp,linking,tips}]`/`pron{lookup,tts}`/`priority(S/A/B)`/`tags`。`theme` が無い項目（文法など）は「テーマなし」グループに入れる。

---

## 1. 全体レイアウト：単語帳カード（左=問題／右=隠れた答え）
- **基本は「問題」と「答え」の2面**。答えは初期状態で隠し、**ボタン1つ（またはカードのタップ／PCはSpace）で表示**。
- レスポンシブ：
  - **PC・iPad横**：左右2ペイン。左=問題、右=答え（初期はブラー＋「答えを見る」ボタン）。
  - **スマホ・縦**：1枚のカード。上=問題、大きな「答えを見る」ボタン→下に答えが開く（フリップ風アニメ可、過度な装飾はしない）。
- 答え表示後に、その場で **SRS採点（忘れた／あいまい／覚えてた）** と「次へ」。
- **発音はいつでも**：問題側・答え側の両方に🔊ボタン。単語は辞書音声/TTS、例文はTTS（既存の発音機能を流用）。答え面では例文の `linking`(‿) と `tips` も表示。

### 問題/答えの中身（カテゴリ別）
- **単語(word)・構文(phrase)**：問題=見出し語（＋文脈として例文を「実際の文章から学ぶ」ために提示。例文中の対象語は空所も選べる）。答え=意味・言い換え・コロケーション・例文(en/jp/linking/tips)・音声。
- **文法(grammar)**：問題=構文名または例文の空所。答え=説明＋例文＋音声。
- **会話(conversation)**：問題=和訳または英文の空所。答え=英文＋音声。
- **想起方向トグル（ボタン）**：`見出し→意味` / `意味→見出し`（EN→JP / JP→EN）。双方向で記憶を強化。
- **学習モード（ボタン）**：`語義` / `例文穴埋め(cloze)`。clozeは例文の対象語を空所にして思い出させる（実文から学ぶ狙い）。

## 2. カテゴリ選択＝常時ボタン（トグルやドロップダウンにしない）
- 画面上部に **常設のボタン群**：`単語` `構文` `文法`（＋データにあれば `会話`）。`type` に対応（単語=word, 構文=phrase, 文法=grammar, 会話=conversation）。
- カテゴリは**常に分離**。選んだカテゴリの項目だけが対象になり、混在させない。
- 現在のカテゴリを明確にハイライト。各カテゴリに**一貫した色**を割り当て（記憶の手がかり／retrieval cue）。

## 3. テーマ範囲ナビ（1–10 / 11–20 …）
- `item.theme` を**10刻みのレンジに自動グルーピング**（1–10, 11–20, 21–30…。存在するテーマから動的生成）。
- 上部バー（PC/iPad）と**ハンバーガーメニュー（スマホ）**の両方から、レンジ→個別テーマへすぐ切替。テーマ名（`themeName`）も表示。
- 「全テーマ（カテゴリ内）」も選べる。テーマ未設定の項目は「その他」。

## 4. 出題セットの量と並び（UIにバランス良く配置）
- **セット量ボタン**：`10` / `30` / `50`（選んだカテゴリ＋テーマ範囲から、その件数を出題）。少数チャンクで負荷を下げる。
- **並び替え**：`ランダム`（毎回シャッフル）と `昇順`（テーマ番号→元の並び順）の切替ボタン。
- これら（カテゴリ／テーマ範囲／セット量／並び）の操作は、学習中も迷わず押せる位置に**一貫配置**（上部ツールバー＋スマホは折り畳み）。
- セットの進捗バー（何枚中何枚）と残り枚数を常時表示。

## 5. 記憶に残すための追加要素（調査に基づく・実装する）
学習科学で効果が確認されている要素を取り入れる。各々を上記UIに自然に統合：
- **アクティブリコール**：答えを隠し自力想起→開く（testing effect）。※本UIの核。
- **間隔反復**：開いた後に3段階採点→次回出題日を調整（spacing effect）。「今日の復習○件」を表示。
- **生成効果**：開く前に「意味を予想」させる小さなプロンプト、または自作例文（既存機能）へ動線。
- **二重符号化**：語＋例文＋**音声**＋テーマ色/小アイコンで多重に記憶。
- **インターリービング**：カテゴリ内で「テーマ横断ランダム」を選べる（似た語の弁別を促進）。
- **チャンク化**：10/30/50（既に実装）。
- **即時フィードバック＆達成感**：採点直後に次へ、セット完了画面、連続学習(ストリーク)表示。
- **苦手フォーカス**：`lapses>=2` か手動★で苦手判定→「苦手だけ」フィルタ。
- **低摩擦操作**：PCはキーボード（Space=答え, 1/2/3=採点, ←→=前後, S=シャッフル, 🔊=P）、スマホはスワイプ（左右=前後 / 上=答え）。
- **ミニマルな1枚集中ビュー**：余計な装飾を排し認知負荷を下げる。ダークモード対応。
- 大きく読みやすいタイポグラフィ、IPA併記、🔊は常に近接配置。

## 6. 厳守
- 既存スタック・規約に合わせ、既存の取り込み/同期/発音/録音機能を壊さない。
- 新ライブラリ追加は理由を添えて確認。アニメは控えめ・60fps・`prefers-reduced-motion`配慮。
- レスポンシブはモバイルファースト。a11y（フォーカス順・aria・コントラスト）。

## 7. 進め方（マイルストーン）
1. ステップ0の現状把握＋UI統合計画を提示（着手前承認）。
2. カードUI（左問題/右答え・1ボタン開示・レスポンシブ）の骨格。
3. カテゴリ常設ボタン（色分け・常時分離）。
4. テーマ範囲ナビ（10刻み・上部＋ハンバーガー）。
5. セット量(10/30/50)＋並び(ランダム/昇順)＋進捗バー。
6. 発音の常時表示（問題/答え両側）と例文linking/tips表示。
7. 想起方向・clozeモード、苦手フィルタ、キーボード/スワイプ。
8. 仕上げ（ダークモード・a11y・モバイル微調整）と受け入れ確認。
各完了時に動作確認手順を添える。判断が要る箇所は勝手に決めず質問。

## 8. 受け入れ基準
1. スマホ/iPad/PCで、左=問題・右(または下)=隠れた答えが**1ボタンで開く**単語帳UIになっている。
2. `単語/構文/文法(/会話)` が**常設ボタン**で選べ、カテゴリは常に分離。色分けされている。
3. テーマが**1–10 / 11–20…のレンジ**で上部とハンバーガーから即切替でき、テーマ名も出る。
4. `10/30/50` 件で出題でき、`ランダム`と`昇順`を切替可能。進捗バーが出る。
5. 問題・答えの両方で**いつでも発音**を再生でき、答えで例文の連結(‿)・tipsが見える。
6. アクティブリコール＋間隔反復が機能し、苦手フィルタ・キーボード/スワイプが使える。
7. 既存の取り込み（`ielts-import.json`）・同期などが従来どおり動く（リグレッションなし）。
