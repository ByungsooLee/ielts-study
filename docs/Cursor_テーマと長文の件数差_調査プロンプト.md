# Claude 調査プロンプト：テーマ単語数と長文ハイライト数が一致しない理由

> このプロンプトを Claude（または Cursor Agent）に貼り付けて、**教材データの設計意図**を調査・説明させる用。
> 実装変更は求めない。まず「なぜこの構成になっているか」を根拠付きで整理してほしい。

---

## 背景

IELTS 学習アプリ（リポジトリ: `ielts-study`）の English「単語」ページでは、テーマを選ぶと次の2つが並んで表示される。

1. **長文（passage）** … テーマの自作英文＋和訳。文中の対象語をハイライト
2. **単語カード** … そのテーマの `type: word` をフラッシュカードで学習

データは `web/public/content/index.json` とテーマ別シャード  
`web/public/content/english/ielts-vocab/theme-{N}.json` に格納。規約は `CLAUDE.md`。

---

## 観測されたギャップ（例：テーマ1「ポリネシアの入植」）

UI ではテーマボタンに **17語** と表示されるが、長文パネルでハイライトされる語は **11語程度** に見える。

`theme-1.json` を機械的に集計すると（2025年時点の静的教材）:

| 区分 | 件数 |
|------|------|
| `items` 全体 | 20 |
| `type: word`（単語カード対象） | **17** |
| `type: phrase`（構文ページ向け） | 3 |
| `passage.sentences[].targets` のユニーク id | **15** |

さらに内訳:

- **長文に一度も登場しない単語**（targets にも無い）: `resolve`, `dog`, `auspicious`（3語）
- **targets にはあるが文中の見出しと表記が一致せずハイライトされない**:
  - `pioneer` ↔ 文中 `pioneers`
  - `seafarer` ↔ 文中 `seafarers`
  - `dub` ↔ 文中 `dubbed`
  - `p-find-one-s-bearings`（phrase）↔ 文中 `find their bearings` など
- フロントのハイライト実装は `item.front` を `\b...\b` で完全一致検索（`PassagePanel.tsx`）

## 調査してほしいこと

以下を、**リポジトリ内のファイル・コメント・生成元データ・ドキュメント**を根拠に説明してください。推測だけで終わらせず、根拠が無い点は「不明」と明記してください。

### A. データモデル全体の意図

1. なぜ 1 テーマに **`items`（単語・構文など）と `passage`（長文）が同居**しているのか？
2. `items` の件数と `passage.targets` の件数は **一致することが前提**なのか、**一致しないことが前提**なのか？
3. `type: word` と `type: phrase` を同一テーマに置く設計意図は何か？（単語ページ vs 構文ページの分離との関係）
4. `index.json` の `themes[].count` と、実際の `items` 件数、UI の `collectThemeStats` の数がズレうる理由は？

### B. 長文（passage）の設計意図

5. `passage` は誰が・どのパイプラインで生成されたか？（`sample/ielts-import.json`, `content-src/ielts-vocab-data.js`, `content:build` 等を確認）
6. なぜ **全単語を長文に入れない**のか？（著作権・文長・学習効果・元の紙教材の構成など）
7. `sentences[].targets` は何を表すか？（その文に「学習上フォーカスする語」／「文中に字面で出る語」／「例文紐づけ」など）
8. `targets` に phrase id を入れる意図は？長文ハイライトや単語ページとの関係は？

### C. 表記ゆれ（活用形・フレーズ）

9. `front`（見出し）と文中表記が異なる場合（pioneers/dubbed 等）は **データ上の仕様**か **生成時の oversight** か？
10. 元の単語帳・教材では「見出し形」と「文中形」の両方をどう扱っていたか？（`content-src/単語マスターリスト.md` 等があれば参照）

### D. UI との関係（説明のみ。実装はまだしない）

11. テーマボタンの件数（17）と長文ハイライト数（11）が違うことは、**ユーザーにとって誤解を招く**か？
12. 意図した学習フローは「長文で文脈 → カードで全語」か「長文に出た語だけ」か？
13. 改善するなら **データ側**（passage に全語を入れる／targets を拡張）と **UI側**（活用形マッチ／件数表示の分離）のどちらが設計に沿うか？
