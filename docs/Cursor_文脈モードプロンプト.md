# Cursor 実装プロンプト：文脈モード（文章から単語を覚える）

> 既存アプリに「文脈モード（passage-first learning）」を追加。元の単語帳の“文章から覚える”構成を引き継ぐ。
> 作り直さず統合。まず現状把握→計画→承認→実装。データ・保管は既存方針（KV `content`）。

## 目的と根拠（語彙習得研究）
単語を孤立提示でなく**一貫した文章の中で**覚える。効果が高い要素を実装に落とす：
- **読みながら聞く（reading‑while‑listening）**：音＋文字で形・意味が定着（読むだけより上、保持も長い）。→ パッセージ/文の🔊再生。
- **タップ即時グロス**：語をタップで日本語の意味/言い換え/例を表示（読みを止めない）。→ 文中ハイライト＋ポップオーバー。
- **入力強調**：対象語をハイライトして気づきを促す。
- **読後の想起テスト**：読んだ後に同じ文章で穴埋め→新文脈へ。「テストがある前提」で読むと定着が上がる。
- **間隔をあけた複数回の再読**：単語帳の「何回読んだ?」と同じ。再読回数を記録し再提示。

注意：純粋な多読だけでは遅い。**読む＋グロス＋想起**を必ずセットにする。

## データ設計（ImportFile に passages を追加・拡張）
`content`（KV）に既存 `items` と並べて `passages` 配列を持たせる（追加キー。無くても既存は動く）。
```ts
interface Passage {
  id: string;            // 例 "t3"
  theme: number;         // テーマ番号（items.theme と対応）
  themeName: string;     // 例 "タイタニック"
  title?: string;
  jp?: string;           // 全体訳（任意）
  sentences: {
    en: string;          // 1文
    jp?: string;
    linking?: string;    // 連結(‿)つき（発音コーチ流用）
    tips?: string[];
    targets?: string[];  // この文に出る対象語の item id（[w-deteriorate ...]）
  }[];
}
// ImportFile: { source, items: StudyItem[], passages?: Passage[] }
```
※ データは生成済み：`ielts-import.json` に `passages`（テーマ1〜10・各3〜5文・targets=item id・linking/tips付き）が同梱されている。`content:push` でそのままクラウドに乗る。
- 対象語ハイライト：文中の語と `targets` の item を対応づけ、タップで item の意味/言い換え/例/発音を表示。
- **原文（書籍そのもの）は使わず、対象語を含む自作パッセージを使用**（著作権配慮・効果は同等）。

## 文脈モードの学習フロー
1. **読む＋聞く**：パッセージ表示。対象語をハイライト。🔊で全文／文ごと再生（reading‑while‑listening）。連結(‿)・tips も表示可。
2. **タップ・グロス**：語をタップ→日本語の意味・言い換え・コロケ・例文・🔊。長押しで「あとで復習に追加」。
3. **音読/シャドーイング（任意）**：既存の録音機能で自分の音声を録る→手本と聞き比べ。
4. **読後テスト（想起）**：同じパッセージの対象語を空所化した穴埋め→できたら新文脈（既存のSRS/言い換えクイズ）へ送る。
5. **再読の記録**：パッセージ単位で「読んだ回数」を記録、間隔をあけて再提示（テーマ範囲ナビから選択も可）。

## SRS・既存機能との連携
- 文脈モードで「復習に追加」/採点した語は既存SM-2スケジュールに反映。
- 発音（辞書/TTS）・録音・テーマ範囲ナビ・カテゴリは既存を流用。
- パッセージは `theme` で items と束ねる（テーマ＝1パッセージ＋その語群）。

## UI
- パッセージビュー：読みやすい大きめ字、対象語は色/下線で強調、タップでグロス。文ごとに🔊と「音読」ボタン。
- 上部に「読む→穴埋め→新文脈」の進行。モバイル対応・キーボード/タップ。
- 文脈モードは学習モード選択（語義／例文穴埋め／類義語／**文脈**）の一つとして追加。

## 厳守
- 既存スタック・データ取得（KV `content`）・SRS・発音・録音を壊さない。
- パッセージ取得失敗時もキャッシュで動作（オフライン耐性）。新ライブラリは理由を添えて確認。
- 書籍原文の転載をしない（自作パッセージのみ）。

## 進め方
1. 現状把握（データ取得層・学習モードの作り）＋統合計画を提示し承認。
2. `passages` の取得・保存（IndexedDBキャッシュ）。
3. パッセージビュー（ハイライト＋タップグロス＋🔊／reading‑while‑listening）。
4. 読後の穴埋め→SRS送り、再読回数の記録。
5. 録音連携・モバイル調整・受け入れ確認。

## 受け入れ基準
1. テーマを選ぶと自作パッセージが表示され、対象語がハイライトされる。
2. 文/全文の🔊再生ができ、語タップで日本語グロス（意味・言い換え・例・発音）が出る。
3. 読後に同パッセージの穴埋めができ、対象語が既存SRSに入る。
4. パッセージの再読回数が記録され、間隔をあけて再提示できる。
5. 既存機能（語義/類義語クイズ・録音・同期）が従来どおり動く。書籍原文は使っていない。
