# 設計書：English / Engineering 統合学習アプリ

> 目的：英語学習とエンジニアリング学習を1つのアプリで。両分野とも「発音・リスニング」まで。
> データはどの端末でも統一表示、Claudeから簡単にアップロード。無料枠で完結。

## 1. 画面構成（IA：情報設計）
- **最上位で2分野に分離**（混ざらない）：`English` / `Engineering` を上部タブ or サイド切替。
- **English** 配下（ごちゃつかないよう4分割）：
  - `単語`（kind=word）／`文法`（grammar）／`構文`（construct：コロケーション・定型表現）／`長文`（passage：文脈モードで読む）
  - （会話は当面 English 内の任意タブ or 構文に内包。将来タブ追加可）
- **Engineering** 配下：
  - `用語`（concept：技術用語を日本語＋英語で）／`設計`（concept：システム設計などを英語で説明できるようにする）
  - ゴール＝「設計や用語を**英語で説明できる**」。各項目に日本語の問い→英語の模範説明→自分で説明（録音）。
- **全分野共通機能**：発音（TTS/辞書）・リスニング（音声再生／ディクテーション／シャドーイング）・SRS復習・テーマ番号ボタン。

## 2. データモデル（統一スキーマ）
4階層：**domain → collection → theme → item**。英語も技術も同じ器に載せる。

```ts
type Domain = "english" | "engineering";
type Kind = "word" | "phrase" | "grammar" | "construct" | "passage" | "concept";

interface Item {
  id: string;            // 全分野で一意・不変（衝突回避にcollectionで名前空間化: 例 "de-sql-join"）
  n: number;             // 通し番号（投入順・SRS新規順）
  domain: Domain;
  collection: string;    // 例 "ielts-vocab" / "de-sql" / "system-design"
  theme: number; themeName: string;
  kind: Kind;
  front: string;         // 英語見出し/用語
  ipa?: string;
  ja: string;            // 日本語の意味/説明
  synonyms?: string[]; collocation?: string;
  examples?: { en: string; ja?: string; linking?: string; tips?: string[] }[];
  audio: { lookup?: string; tts: string; };   // ← 発音・リスニングを全kind共通で実現
  explain?: { prompt_ja: string; model_en: string };  // engineering: 英語で説明する練習用
  tags?: string[]; priority?: "S"|"A"|"B"; links?: string[]; note?: string;
}
```
- ポイント：**どの kind も `audio.tts` を持つ**ので、発音・リスニングは分野を問わず同じ仕組みで動く。
- 技術学習は `kind:"concept"` ＋ `explain`（日本語の問い→英語模範）で「英語で説明する」練習に対応。

## 3. ストレージ設計（無料枠・結論）
**教材＝静的JSONシャード on Pages（CDN）／進捗＝KV／録音・音声キャッシュ＝端末IndexedDB。専用RDBは今は不要。**

理由：教材は「読み取り中心・全端末で同一・追記更新」なので、CDN静的配信が**最安・最速・実質無制限**。RDBは「サーバ側で検索・集計」が要るとき初めて必要になる道具で、現状その要件はない。

### 無料枠の比較（1人利用はいずれも余裕。用途適合で選ぶ）
| 選択肢 | 無料枠の目安 | 役割適合 |
|---|---|---|
| **Pages 静的配信** | 配信実質無制限（ビルド 月500回） | ◎ 教材（読み取り中心・全端末同一） |
| **Workers KV** | 読み10万/書き1千/日・1GB | ◎ 進捗（個人・小さく頻繁更新・端末同期） |
| D1 (SQLite) | 行読み500万/書き10万/日・5GB | △ 将来「サーバ側検索/集計」が要る時 |
| Supabase(Postgres) | 500MB・7日無操作で一時停止 | △ 外部・今回は過剰 |
| Firebase Firestore | 5万読/2万書/日・1GB | △ 外部・今回は過剰 |

→ **結論：Pages静的（教材）＋ KV（進捗）の二本立て。** 検索要件が出たら D1 を後付け。

### 配置（CDN）
```
web/public/content/
  index.json                         ← 目次（domain/collection/theme/件数/version）
  english/ielts-vocab/theme-1.json … ← 英語・単語帳（既存206語＋passages）
  english/.../...                    ← 文法/構文/長文も同形
  engineering/de-sql/sql-basics.json ← 技術（concept）
  engineering/system-design/...
```
`index.json` を最初に取得（軽い）→ 選んだ theme のファイルだけ遅延取得→ `version` 変化時のみ再取得（CDN/ETagで高速）。

## 4. Claudeからのアップロード（統一・簡単）
1. Claudeが対象の `domain/collection/theme` JSON と `index.json` を生成。
2. リポジトリ `web/public/content/...` にミラー（Claudeが直接書込み）。
3. `git push origin main` → GitHub Actions が Pages へデプロイ → CDN反映。
4. 全端末は次回起動で目次を見て差分取得＝**どの媒体でも統一表示**。
- 既存の自動反映（`content.yml`/`sync-to-app.sh`）をこの構成に更新。進捗(KV)は従来どおり。

## 5. 発音・リスニング（両分野共通）
- 発音：単語＝辞書の実音声優先、文＝Worker経由TTS。アクセント en-GB/US/AU 切替。生成音声は端末キャッシュ。
- リスニング：item/passage/explain.model_en の音声再生、**ディクテーション**（聞いて入力）、**シャドーイング**（録音して手本と比較）。
- 技術学習：用語・設計の「英語の説明文」をTTS→聞く／真似る／自分で英語で説明（録音）。これで「英語で説明できる」を鍛える。

## 6. 移行
- 既存 `ielts-import.json`（206語＋passages）を `english/ielts-vocab/theme-*.json` に分割し `index.json` 生成。
- `md2json.js` を「domain/collection/theme でシャード＋目次出力」に拡張。番号台帳(number-ledger)は流用。
- 進捗(KV)は id 単位なので影響なし（id は今後 collection 名前空間で衝突回避）。
