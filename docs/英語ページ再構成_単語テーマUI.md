# 実装プロンプト：英語ページの再構成（ヘッダ整理＋単語ページをテーマ選択UIに）

> English ページのIAを分かりやすくする。作り直さず統合。まず現状把握→計画→承認→実装。
> データは `web/public/content/index.json`＋テーマ別シャード（各 theme ファイルに `items` と `passage` あり）。規約は `CLAUDE.md`。

## 1. ヘッダ（English 内ナビ）— 確定版
- **削除**：`ライブラリ`、`会話`、**`構文`タブ**（phrase は単語テーマ内に統合）。
- **現行ヘッダ（5項目）**：`単語` / `文法` / `あいまい一覧` / `類義語クイズ` / `設定`。
- 構文（type=phrase）は各テーマの長文ハイライト＋語彙カードに含まれる。構文だけ学びたい場合はテーマ学習ビュー内の **「構文のみ」サブフィルタ** を使う。

## 2. 単語ページを「テーマ選択UI」に全面変更
- **入口＝テーマ選択**：`index.json` の english/ielts-vocab の themes からテーマ番号ボタン（番号＋themeName＋件数）。未ロードでも index から件数表示。テーマが増えたら `1–10 / 11–20` レンジ切替。
- **教材取得（lazy）**：起動時は `index.json` のみ。テーマ選択時にその theme ファイルを取得し、IndexedDB に version キーでキャッシュ。version 変化時のみ再取得。選択後は同コレクションの他テーマをバックグラウンド prefetch。KV `/content` はフォールバックとして当面維持。
- **テーマを選ぶ→そのテーマの学習ビュー**：
  - **長文（passage）**：`passage` 表示（英文＋和訳、対象語ハイライト `{id,text}`、文ごと🔊、linking/tips）。
  - **語彙カード**：そのテーマの `items`(type=word/phrase) をフラッシュカード学習。**全件出題**（枚数選択なし）。出題モード（見出し→意味／意味→見出し／例文穴埋め）、並び(ランダム/昇順)、単語/構文サブフィルタ、SRS採点はビュー内。
- 旧『長文』タブ・『構文』タブは廃止し、テーマ内に統合。

## 3. 各ヘッダ項目の中身
- **単語**：テーマ選択→テーマ学習（長文＋word/phrase カード）。
- **文法**：type=grammar（theme-0 シャードを lazy 取得）。
- **あいまい一覧**：SRS あいまい/苦手の横断一覧。即学習・🔊。
- **類義語クイズ**：既存のまま（テーマ別）。
- **設定**：同期、Worker URL、アクセント等。

## 4. 共有・厳守
- SRS / 発音 / 録音 / 進捗(KV) は既存流用。
- データ駆動（テーマ/件数/passage は index＋シャードから）。
- KV 教材は静的配信が本番で安定確認できるまで維持。確認後に `content:push` / `content.yml` を段階廃止。

## 5. 受け入れ基準
1. English ヘッダが `単語/文法/あいまい一覧/類義語クイズ/設定` の5項目。
2. 単語ページはテーマ選択UI（index から件数表示、選択時 lazy 取得）。
3. テーマ内で長文＋word/phrase カードを学べる。構文のみフィルタ可。
4. あいまい一覧・類義語・SRS・発音・同期は従来どおり。
5. モバイルで答え表示時にテキストが見切れない（折り返し）。
